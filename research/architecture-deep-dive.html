Generative Augmentation Architectures

R-GAN
Regularized + Spectral Norm

492 fraud samples is too few. R-GAN synthesizes more to balance LightGBM training.

Key architectural choices
  • spectral_norm wraps every nn.Linear — constrains the discriminator's Lipschitz constant, preventing training collapse on tabular data
  • Generator: 128→256→512→256→50 — expands into richer space then compresses to feature space
  • No BatchNorm in discriminator — required by WGAN-GP gradient penalty math
  • n_critic=5 — discriminator runs 5 steps per generator step
Fraud-specific design
  • lambda_sim=0.5: Similarity Measure Loss penalizes generator if synthetic fraud doesn't match real fraud mean & std — directly targets minority-class quality
# rgan.ipynb — Cell 10 def spectral_norm(module): return nn.utils.spectral_norm(module) self.net = nn.Sequential( spectral_norm(nn.Linear(128, 256)), nn.BatchNorm1d(256), nn.CELU(), spectral_norm(nn.Linear(256, 512)), ... )
T-GAN
Transformer Generator

Can self-attention capture the complex correlations between 51 PCA features better than MLP?

Key architectural choices
  • Noise vector z projected to 8 × 256 tokens — tabular features treated as a sequence so attention can relate them to each other
  • nhead=8 in generator, nhead=4 in discriminator — shallower discriminator prevents it from overpowering generator
  • 4 Transformer encoder layers in generator, 2 in discriminator
Implementation constraint
  • Flash attention disabled (enable_flash_sdp(False)) — flash kernels don't implement the backward pass needed for WGAN-GP gradient penalty on this PyTorch build
# tgan.ipynb — Cell 16 self.input_proj = nn.Linear( cfg.latent_dim, cfg.seq_len * cfg.d_model # 8×256 ) x = self.input_proj(z).view( -1, self.seq_len, self.d_model ) x = self.transformer(x) # 4-layer encoder
CTGAN
Tabular, VGM-Conditioned

Amount has a skewed, multi-modal distribution — standard GANs fail here. VGM conditioning solves it.

Key architectural choices
  • Variational Gaussian Mixture pre-processes continuous columns into mixture components; the mode is passed as a conditional vector to the generator
  • pac=10 — PAC-GAN groups 10 samples per discriminator forward pass for tabular stability
  • n_epochs=800 vs R-GAN's 300 — more epochs needed to fit the conditional distribution
Library vs from-scratch
  • Uses sdv-ctgan library — VGM is an internal preprocessing component. Configuration is declarative rather than architectural
# ctgan.ipynb — Cell 5 cfg.embedding_dim = 128 cfg.generator_dim = (256, 256) cfg.discriminator_dim = (256, 256) cfg.pac = 10 # PAC-GAN grouping cfg.n_epochs = 800 # conditional fit
All three models feed synthetic fraud samples into a downstream LightGBM classifier trained on 250,170 records. The GAN's job is purely augmentation — LightGBM does the final classification.

Reconstruction-Based Detection Architectures

GANomaly
Dual Encoder-Decoder

Train only on normal transactions. Fraud = whatever doesn't compress and reconstruct consistently.

Key architectural choices
  • Three modules: Encoder₁ → Decoder → Encoder₂. Anomaly score = ‖z − ẑ‖₂ — the gap between the two latent codes
  • All three trained jointly under one optimizer: Adam(E1 + D + E2 params)
  • Loss weights: w_con=50, w_adv=1, w_enc=1 — reconstruction dominates; GAN is a regularizer
Threshold (no fraud labels)
  • 95th percentile of validation normal scores — completely unsupervised, no peeking at the 492 fraud labels
# ganomaly.ipynb — Cell 10 opt_gen = Adam( list(encoder_1.parameters()) + list(decoder.parameters()) + list(encoder_2.parameters()) ) # anomaly score at inference: score = ‖encoder_1(x) − encoder_2(x̂)‖₂
Skip-GANomaly
+ U-Net Skip Connections

Fraud may differ only in 2-3 subtle features — a bottleneck of 32 dims can wash that out. Skip connections preserve it.

Key architectural choices
  • Encoder saves intermediate activations at every layer; decoder receives them as residual additions — U-Net pattern on tabular data
  • 4-level encoder: 29→256→128→64→32; decoder mirrors it in reverse
  • Anomaly score blends two signals: 0.9 × reconstruction + 0.1 × discriminator feature mismatch
Why this matters for fraud
  • Skip connections let the decoder directly reference pre-bottleneck activations — subtle feature-level anomalies survive the latent compression
# skip-ganomaly.ipynb — Cell 15 def forward(self, x): skip_outputs, h = [], x for layer in self.layers: h = layer(h) skip_outputs.append(h) return h, skip_outputs # pass skips z, skips = encoder(x_real) x_recon = decoder(z, skips)
f-AnoGAN
Fast Encoder Inference

Standard AnoGAN requires iterative gradient optimization per sample at inference — too slow for real-time fraud scoring.

Two-phase training
  • Phase 1 (100 epochs): Train WGAN-GP Generator + Discriminator on normal transactions
  • Phase 2 (50 epochs): Freeze G & D. Train a separate Encoder to map any input directly into the GAN's latent space
Encoder loss (Phase 2)
  • enc_loss = loss_rec + κ × loss_feat — matches both pixel-level reconstruction and discriminator's internal feature representations
  • Anomaly score uses z-score normalization (not MinMax) — more robust to extreme outlier scores
# fanogan.ipynb — Cell 11 (Phase 2) z_hat = E(x_real) # single forward pass x_hat = G(z_hat) loss_rec = MSE(x_hat, x_real) loss_feat = MSE(D(x_hat).feat, D(x_real).feat) enc_loss = loss_rec + κ * loss_feat
Pipeline B models are semi-supervised — trained exclusively on 284,315 normal transactions, no fraud labels used. Fraud is an emergent signal at inference, not a training target.

How Architecture Choices Address the Core Problem

Shared design decisions across all 6 models
29–51 features (V1-V28 + Amount, Time excluded) WGAN-GP gradient penalty for training stability No Tanh on output — real features are unbounded No BatchNorm in critic — required by WGAN-GP math Holdout evaluated at natural 0.172% fraud rate Threshold from validation normals only
Pipeline A — What each architecture adds
  • R-GAN: Spectral norm stabilizes training + Similarity Loss ensures synthetic fraud is statistically close to real fraud — addresses quality of augmentation
  • T-GAN: Self-attention over feature tokens captures multivariate correlations — addresses realism of generated distributions
  • CTGAN: VGM conditioning handles multi-modal Amount distribution — addresses feature heterogeneity in tabular data
Pipeline B — What each architecture adds
  • GANomaly: Second encoder creates an anomaly score from latent space inconsistency — fraud cannot be consistently re-mapped
  • Skip-GANomaly: U-Net skip connections preserve subtle feature deviations that the bottleneck would otherwise erase — catches low-magnitude anomalies
  • f-AnoGAN: Pre-trained encoder replaces iterative optimization — makes anomaly scoring viable in production (single forward pass)
